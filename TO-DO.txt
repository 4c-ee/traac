# traac TO-DO List

## Code Quality & Refactoring
- [ ] **UI/Logic Separation**: Split `src/ui/app.rs` (currently >700 lines) into smaller, more manageable modules (e.g., `state.rs`, `view.rs`, `update.rs`).
- [ ] **Structured Error Handling**: Replace `Box<dyn std::error::Error>` and string-based errors with a structured error type using `thiserror`.
- [ ] **Config Path Management**: Fix `Config::save()` to respect the custom path used during `Config::load()`, rather than always defaulting to `~/.config/traac/config.toml`.
- [ ] **Remove Dead Code**: Remove or integrate `src/ui/application.rs` if it's no longer used.
- [ ] **Color Parsing Optimization**: Parse color strings from the config once into `iced::Color` and store them in the `App` state, instead of parsing them on every `view()` call.
- [ ] **Consistent Async Interface**: Refactor `LastFm` and `mpris` modules to use a more consistent async pattern across the board.

## Performance & Optimization
- [ ] **Minimize Polling**: Reduce reliance on the 5-second `Tick` for track updates by better leveraging MPRIS D-Bus events. Ensure `Tick` is only used for time-elapsed scrobble checks.
- [ ] **Efficient Tray Handling**: Replace the manual tray thread/sleep loop with a more native async integration if possible.
- [ ] **Image Cache**: Implement a simple cache for album art to avoid re-fetching or re-processing the same images frequently.

## Feature Requests
- [ ] **Multi-Player Support**: Allow users to see and switch between multiple active MPRIS players instead of just picking the first one.
- [ ] **Scrobble History**: Add a view to see recently scrobbled tracks.
- [ ] **Lyrics Integration**: Fetch and display lyrics for the currently playing track.
- [ ] **Interactive Config UI**: Add a dedicated settings view within the app to edit `config.toml` options (colors, position, ignored players) visually.
- [ ] **Systemd/Auto-start**: Provide a template for a systemd user service or integration with desktop auto-start.
- [ ] **More UI Themes**: Include pre-set color schemes (e.g., Nord, Gruvbox) in addition to the default Catppuccin.
- [ ] **Compact Mode**: A "mini-player" mode that only shows the track title and artist in a very small window.

## Bugs & Minor Improvements
- [ ] **Documentation**: Update README with anything else relevant to the user; how to initialize tray apps, etc.
- [ ] **Precision in Duration**: Ensure `mpris` duration calculations (microseconds to seconds) don't cause issues with extremely short tracks.
- [ ] **Window Visibility**: Verify if returning an empty/zero-size container in `view()` when hidden is the most efficient way to hide the layer shell window.
- [ ] **Notification Content**: Add album art to desktop notifications if available.
- [ ] **Sanity Check Feedback**: Provide UI feedback when a track is being "sanity checked" or if it fails the check.
